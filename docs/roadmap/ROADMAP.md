# Development Roadmap

## Phase 0: Planning

- Finalize product scope.
- Finalize architecture.
- Finalize database schema.
- Finalize user flows.
- Finalize first experiment SOP.
- Define MVP acceptance criteria.

## Phase 1: Core MVP

- Set up frontend project.
- Set up Supabase project.
- Add authentication.
- Add student profile.
- Build hostel room scene.
- Build campus path scene.
- Build department entrance scene.
- Build laboratory scene.
- Add basic player controller.
- Add basic interactions.
- Add PPE and attendance gate.
- Build Agarose Gel Electrophoresis experiment.
- Add mistake tracking.
- Add contamination tracking.
- Add attempt report.
- Add faculty dashboard MVP.

## Phase 2: Training And Progression

- Add safety training modules.
- Add instrument training modules.
- Add XP system.
- Add career levels.
- Add unlock rules.
- Add certificates.

## Phase 3: Realism And Immersion

- Add improved NPC schedules.
- Add more ambient audio.
- Add better avatar animations.
- Add richer lab instrument interactions.
- Improve contamination simulation.
- Optimize 3D performance.

## Phase 4: Experiment Expansion

- DNA extraction
- PCR
- ELISA
- Cell culture
- Multiple laboratories
- Additional instruments

## Phase 5: Institute SaaS

- Institute dashboard
- Multi-campus support
- Bulk import students
- SOP upload
- Custom lab templates
- Institute analytics

## Phase 6: Multiplayer

- Presence system
- Shared lab sessions
- Collaborative experiments
- Voice or text chat
- Instructor-led live training

## MVP Acceptance Criteria

- Student can log in.
- Student starts in hostel room.
- Student can move to campus, department, and lab.
- Student cannot enter lab without PPE and attendance.
- Laboratory feels active.
- At least one NPC movement loop exists.
- One experiment can be completed end to end.
- Wrong actions are recorded.
- Contamination affects result quality.
- Faculty can view attempt results.
